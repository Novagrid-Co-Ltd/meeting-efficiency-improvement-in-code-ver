## 2. 処理フロー詳細（8分）

### Step 1: Docs取得
- Google Docs APIでドキュメント取得
- Geminiの文字起こしタブからトランスクリプト抽出
- eid（会議ID）を文書内から取得

### Step 2: Calendar照合
- Google Calendar APIで過去14日間のイベント取得
- eidでイベントをマッチング
- 参加者リスト・開始/終了時刻を取得

### Step 3: データ保存（Supabase）
- `row_meeting_raw`: 生データ（文字起こし、イベント情報）
- `tf_meeting_attendee`: 参加者リスト（person_identity照合済み）
- `tf_individual_score_input`: 個人評価用入力データ
- すべてUPSERT（冪等性あり、再実行可能）

### Step 4: LLM会議評価
- プロンプトバージョン: `meeting-eval-v1.1.0`
- LLMプロバイダー: OpenAI（環境変数で切替可能、Geminiも対応）
- レスポンスをJSON解析 → `out_meeting_eval` に保存

### Step 5: LLM個人評価
- プロンプトバージョン: `individual-eval-v1.1.0`
- 参加者ごとに**順次実行**（APIレート制限対策）
- 1人失敗しても残りは継続 → `out_individual_eval` に保存

### Step 6: レポート生成
- HTML/テキスト両方生成
- レーダーチャート: QuickChart.io API経由でPNG URL生成
- レスポンスに全レポートを含めてn8nに返却

## 3. 6軸スコア定義と設計意図（7分）

### 会議評価 6軸（1〜5点）

| 軸 | 定義 | 評価のポイント |
|----|------|---------------|
| 目的の明確さ | アジェンダが事前共有され目標が明確か | ゴールの具体性 |
| 意思決定 | 議題に対し適切な意思決定がなされたか | 決定プロセスの質 |
| TODO明確化 | 担当者・期限付きのアクションが出たか | 「誰が何をいつまで」の有無 |
| 役割明確さ | ファシリテーター・議事録係が機能したか | 会議運営体制 |
| 時間効率 | 脱線や繰返しなく効率的に進行したか | 時間の使い方 |
| 発言バランス | 全員に発言機会があったか | 特定人物への偏り |

### 個人評価 6軸（1〜5点）

| 軸 | 定義 | 評価のポイント |
|----|------|---------------|
| 課題理解度 | 議題の本質を捉えた的確な発言か | 表面的でなく根本原因まで |
| 発言価値密度 | 発言が議論に実質的価値を与えたか | 量より質 |
| 構造的思考 | 論理的整理と優先度付けができているか | 因果関係の明確さ |
| 協調的影響力 | 他者の意見を活かし議論を高められたか | チーム貢献度 |
| 意思決定推進 | 議論を決定に向けて前進させたか | 堂々巡り打破 |
| 実行連携度 | 具体的アクションに繋げる提案をしたか | 「次何する」の明確化 |

### プロンプト設計の意図
- **具体性重視**: 抽象的表現を禁止、文字起こしからの引用を必須化
- **エビデンス付き**: 個人評価は3件以上の発言引用が必須
- **日本語出力**: プロンプト・出力ともに日本語
- **JSON出力のみ**: 余計なテキストを含めない指示で解析安定性を確保

## 5. エラー時の挙動（3分）

### LLM評価失敗時
- `evaluation_status: "failed"` でSupabaseに保存（スコアはすべてnull）
- **生のLLMレスポンスは保存される**（デバッグ用）
- 会議評価失敗 → レポートはスコアなしで生成
- 個人評価失敗 → **その人だけスキップ、他の人は継続**

### API呼び出しエラー時
- `AppError` でコード・ステップ・メッセージを構造化返却
- 管理者メール通知（`ADMIN_EMAIL` 設定時）
- n8n側でエラーハンドリング可能

### 主なエラーコード
| コード | 原因 |
|--------|------|
| DOCUMENT_NOT_FOUND | Google Docs fileIdが不正 |
| CALENDAR_NOT_MATCHED | カレンダーイベントが見つからない |
| MISSING_FILE_ID | リクエストにfileIdがない |
| UNAUTHORIZED | API Keyが不正 |
